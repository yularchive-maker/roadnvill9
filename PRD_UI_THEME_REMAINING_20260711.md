# roadnvill9 UI/UX 잔여 작업 PRD

최종 업데이트: 2026-07-11

## 1. 목적

roadnvill9는 고객용 공개 사이트가 아니라 내부 직원용 예약, 운영, 정산 관리 웹앱이다.
현재 기능은 계속 누적되어 왔지만 화면 밀도와 디자인 일관성이 떨어져 있어, 다음 단계는 기능 추가보다 UI/UX 정리와 사용 흐름 단순화에 집중한다.

이 문서는 라이트/나이트 버전 작업과 아직 남은 UI/UX 작업을 한곳에 정리하기 위한 기준 문서다.

## 2. 작업 원칙

- 기존 기능과 Supabase 데이터를 깨뜨리지 않는다.
- soft delete 방식을 유지한다.
- 기존 마이그레이션 파일 흐름을 유지한다.
- 환경변수와 비밀키는 출력하거나 커밋하지 않는다.
- 수정 전 관련 파일을 확인하고, 수정 후 `npm run build` 또는 동등한 빌드 검증을 한다.
- UI 변경은 먼저 로컬에서 확인하고, 사용자가 승인하면 배포한다.
- 보안상 서버 검증, 인증, RLS, 비밀키 보호 작업은 이미 적용한 방향을 유지한다.

## 3. 최근 완료된 보안 작업

- 공개 anon SELECT 정책 정리 SQL 작성 및 적용 완료.
- 서버 API 입력 검증과 필드 화이트리스트 적용.
- API 응답 과다 노출 축소.
- 관리자 전용 API 보호 강화.
- Telegram webhook 시크릿 검증 유지.
- XSS 직접 HTML 렌더링 점검 완료.
- CSP와 기본 보안 헤더 적용.
- PII 로그 축소 및 `.gitignore`, `.vercelignore` 보강.
- API 에러 원문 노출 축소.

## 4. 앞으로 해야 할 주요 작업

### 4.1 라이트/나이트 테마

상태: 미완료

목표:
- 현재 다크 중심 디자인을 유지하되, 라이트 버전도 선택 가능하게 만든다.
- 사용자가 선택한 테마를 브라우저에 저장한다.
- 사이드바 또는 상단 헤더에 테마 전환 버튼을 둔다.

요구사항:
- CSS 변수 기반으로 색상 체계를 정리한다.
- 다크/라이트 모두에서 표, 모달, 입력창, 캘린더, 타임테이블 가독성을 확인한다.
- 텍스트 대비가 낮은 색상은 조정한다.

### 4.2 전체 UI 디자인 일관화

상태: 미완료

목표:
- 대시보드, 예약관리, 기준정보, 정산, 타임테이블의 카드/버튼/입력창/모달 스타일을 통일한다.
- 내부 운영툴답게 과한 장식보다 읽기 쉬운 밀도와 정렬을 우선한다.

요구사항:
- 버튼 높이, border radius, 색상, hover 상태 통일.
- 금액은 우측 정렬.
- 긴 드롭다운은 짧고 조작하기 쉬운 형태로 조정.
- 모바일에서 텍스트가 겹치지 않게 확인.

### 4.3 담당자 전달사항 위치와 동작

상태: 일부 완료, 최종 확인 필요

현재 방향:
- 대시보드 하단 전체 목록 방식은 제거.
- 좌측 `상세 검색` 아래에 `메모 n건`, `완료 n건` 형태로 표시.
- 텍스트를 누르면 메모/완료 목록이 팝업으로 뜬다.
- 메모 숫자는 눈에 띄도록 붉은색 계열로 표시.

확인할 것:
- 메모 입력 위치가 사용자 의도와 맞는지.
- 긴급에서 완료 후 체크 해제 시 일반으로 잘못 돌아가는 문제가 재발하지 않는지.
- 다른 계정 로그인 시 긴급 팝업 1회 표시 흐름이 유지되는지.

### 4.4 예약 등록 간소화

상태: 미완료

목표:
- 예약 등록 화면이 너무 복잡하므로 입력 흐름을 줄인다.

확정된 방향:
- 고객결제처 첫 번째 구분: `플랫폼 / 여행사`
- 고객결제처 두 번째 구분: 플랫폼명 또는 여행사명
- 두 번째 구분에서 선택된 이름을 유입처 기본값으로 자동 입력한다.
- 플랫폼/여행사 기준정보에 등록된 개인/단체 수수료를 예약 등록 시 자동 반영한다.
- 특정 예약 건만 수동 변경할 수 있는 예외 입력은 유지한다.

함께 확인할 버그:
- 기준정보 > 플랫폼/여행사에서 플랫폼 `길과마을` 삭제가 안 되는 문제.

### 4.5 정산요약 월별 보기

상태: 일부 완료 가능성 있음, 최종 확인 필요

목표:
- 정산요약에서 월별로 정산 내역을 볼 수 있게 한다.
- 월 선택 UI는 현재 웹앱 디자인과 통일한다.

확인할 것:
- 월 선택 드롭다운이 너무 길지 않은지.
- 금액 컬럼은 우측 정렬인지.
- 1일~말일까지의 월 단위 데이터가 정확히 반영되는지.

### 4.6 타임테이블 디자인 개선

상태: 진행 중/불안정, 우선순위 높음

목표:
- 현재 타임테이블 내용 구조는 유지하되, 디자인만 더 직관적으로 개선한다.
- 구역별, 예약별, 업체별 분류는 유지한다.
- `패키지별` 표현은 단품/비패키지 예약도 있으므로 `예약별` 중심으로 정리한다.

확정된 방향:
- 일/주/월 보기 유지.
- 전체 보기에서 월 단위는 각 체험 프로그램을 전부 나열하지 않고 예약 건별로 보여준다.
- 예약자명, 인원, 예약번호 등 핵심 정보가 먼저 보이게 한다.
- 텍스트 클릭 시 상세 팝업을 띄운다.
- 같은 예약 건 안의 여러 체험 프로그램은 시간 겹침으로 표시하지 않는다.
- 다른 예약끼리 시간/장소가 겹칠 때만 형광연두 테두리 등으로 표시한다.

추가 요구:
- 1박2일 예약은 다음날까지 이어져 보이게 한다.
- 예: 이순신고교 예약이 24일~25일이면 두 날짜 칸이 하나의 예약 흐름처럼 연결되어야 한다.
- 타임테이블 격자 위에 투명 레이어를 올려 자유롭게 이어지는 형태로 표현하는 방식을 검토한다.

주의:
- 박근아 7월 4일 건처럼 하나의 예약 안에 있는 프로그램이 여러 예약처럼 보이면 안 된다.
- 금소시네마처럼 19:00~19:30 일정만 있는데 오전에도 중복 표시되는 데이터 매핑 문제를 확인해야 한다.

### 4.7 숙박 객실 주중/주말 금액

상태: 설계만 완료, 구현 미완료

목표:
- 숙박 객실 기준정보에서 주중 금액과 주말 금액을 분리한다.
- 금, 토, 일 중 어떤 요일을 주말가로 볼지 객실별로 선택 가능하게 한다.
- 예약 등록에서 숙소를 선택할 때 예약 날짜 기준으로 적용 금액을 자동 표시한다.

결정된 방향:
- 기존 `금액`, `주중금액`, `주말금액` 3개 구조는 중복될 수 있다.
- 기본 금액 하나에 의존하기보다 주중/주말 금액과 주말 적용 요일 선택을 명확히 둔다.
- 특정 예약 건은 수동 변경 가능해야 한다.

필요 작업:
- DB 마이그레이션 설계.
- 기준정보 UI 수정.
- 예약 등록 숙소 선택 로직 수정.
- 기존 데이터 마이그레이션 방식 결정.

## 5. 우선순위 제안

1. 라이트/나이트 테마 뼈대 만들기
2. 전체 UI 디자인 변수 정리
3. 예약 등록 간소화
4. 타임테이블 디자인 안정화
5. 정산요약 월별 보기 최종 확인
6. 숙박 객실 주중/주말 금액 설계 및 DB 마이그레이션
